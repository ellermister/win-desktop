## 只用 Go 不行，需要 C/C++ 编译器

webview_go 在 Windows 上通过 **CGO** 编译 C++ 代码（`webview.h` 里是 C++ + WebView2），所以：

- **必须**安装 **C++ 编译器**，否则 `go build` 会失败（可能表现为 “build constraints exclude all Go files” 或 CGO 报错）。
- Windows 上常见两种方式：
  1. **MinGW-w64（g++）**  
     - 安装 [MSYS2](https://www.msys2.org/)，在 MSYS2 里执行：`pacman -S mingw-w64-ucrt-x86_64-gcc`  
     - 把 MinGW 的 `bin` 目录（例如 `C:\msys64\ucrt64\bin`）加入 PATH，保证 `g++` 在命令行可用。
  2. **MSVC（Visual Studio）**  
     - 安装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) 或完整 Visual Studio，勾选 “使用 C++ 的桌面开发”。  
     - 在 “x64 Native Tools Command Prompt” 或 “Developer PowerShell” 里执行 `go build`。

装好编译器后，在项目目录执行：

```bash
cd e:\projects\win-desktop
go build -ldflags="-H windowsgui" -o win-desktop.exe .
```

**若在 MSYS2 的 UCRT64 终端里编译**，需要显式指定目标为 Windows，否则会报 “build constraints exclude all Go files”：

```bash
cd /e/projects/win-desktop
GOOS=windows go build -ldflags="-H windowsgui" -o win-desktop.exe .
```

即可用**本地已修改的 webview_go** 编出支持 **HTML 背景透明** 的窗口程序。

## 使用 w64devkit（绿色版 MinGW）时出现 “cannot parse _cgo_.o as ELF, Mach-O, PE”

**原因**：w64devkit 默认让汇编器生成 **BigObj COFF** 格式，而当前 Go 的 CGO 无法解析这种格式（[golang/go#24341](https://github.com/golang/go/issues/24341)、[w64devkit#290](https://github.com/skeeto/w64devkit/issues/290)）。

**可以**用 w64devkit，但需要二选一：

1. **换用 MSYS2 的 MinGW**（推荐，同样可便携）  
   - 安装 [MSYS2](https://www.msys2.org/)（可装到 U 盘或单独目录），在 MSYS2 终端执行：  
     `pacman -S mingw-w64-ucrt-x86_64-gcc`  
   - 把 `C:\msys64\ucrt64\bin`（或你的 MSYS2 安装路径下的 `ucrt64\bin`）加入 PATH，**且排在 w64devkit 的 bin 前面**，再执行 `go build`。  
   - 这样 CGO 会用 MSYS2 的 gcc/g++，不会生成 BigObj，即可正常编译。

2. **自建一份“无 BigObj”的 w64devkit**  
   - 从 [w64devkit 源码](https://github.com/skeeto/w64devkit) 自行构建，在构建脚本/ Dockerfile 里**去掉**那行给汇编器默认加 `-mbig-obj` 的 sed/补丁（见 issue #290），再使用你构建出的工具链。  
   - 用这份工具链的 `bin` 做 PATH，再 `go build` 即可。
